Technical Interview Cheat Sheet

🚀 Core Advanced RAG Concepts

Q: Why did you use Hybrid Search (FAISS + BM25)?
"Vector search (FAISS) captures semantic meaning but often misses specific keywords like subject codes (e.g., 'AC-101') or specific dates. BM25 ensures exact keyword matching. Combining them via Reciprocal Rank Fusion (RRF) provides the best of both worlds: context awareness and keyword precision."
Q: Explain your RRF (Reciprocal Rank Fusion) logic.
"I used the formula: 1 / (60 + rank). I gave semantic results a 0.7 weight and keyword results 0.3. This prioritizes meaning while ensuring that if a specific term is found via keyword search, it still influences the final top-5 chunks."
Q: Why Query Decomposition/Multi-Query?
"User queries are often multi-faceted (e.g., 'What are the fees AND the attendance rules?'). A single vector search might only find chunks for one part. My query_processor breaks this into sub-questions, retrieves for each, and fuses the results to ensure 100% coverage."

🛠 Implementation Specifics

Q: Why pdfplumber over pypdf?
"University documents are table-heavy. Standard parsers often mash table rows into a single string, losing structural relationships. pdfplumber preserves whitespace and visual alignment, which is critical for the LLM to understand data like fee structures or exam schedules."
Q: How do you handle "Inappropriate Language" or "Topic Changes"?
"I implemented a Manager LLM step in the query processor. It flags inappropriate input before retrieval to save costs. It also detects topic drift to clear the conversation history, preventing 'context contamination' where old info confuses new answers."

📊 Evaluation (The "Senior" Part)

Q: How do you know this system is actually better?
"The system is designed for RAGAS Evaluation. By outputting structured JSON (answer, context, source), we can measure: 1. Faithfulness: Is the answer derived solely from the provided PDF? 2. Answer Relevancy: Does the answer actually solve the student's problem?"
