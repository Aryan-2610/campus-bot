CampusBot: Project Architecture

A modular, production-grade Advanced RAG system designed for academic document intelligence.

📂 Root Folder: campus-assistant/

app.py
The Gatekeeper. A Flask web server that serves the DTU clone and provides a /chat POST endpoint for the widget.
engine.py
The Conductor. Orchestrates the entire flow: sends input to query expansion, gets context from retrieval, and generates the final JSON response.
query_processor.py
The Brain. Uses LLM-based logic to detect topic changes (safety) and perform Query Decomposition (breaking complex questions into sub-parts).
retrieval.py
The Librarian. Handles PDF parsing via pdfplumber and manages the Hybrid Search (FAISS + BM25) with RRF scoring.
prompts.py
The Instruction Manual. Centralizes all LLM system prompts, JSON schemas, and expansion templates. Keeps logic clean.
.env
The Vault. Stores sensitive GOOGLE_API_KEY.

🎨 Frontend & Data

static/widget.js
The UI. A vanilla JS floating widget that handles the chat bubbles, formatting, and asynchronous API calls.
templates/index.html
The Host. A localized clone of the DTU website where the chatbot lives.
data/
The Source. Folder containing the university PDFs (policies, schedules) used for RAG grounding.
